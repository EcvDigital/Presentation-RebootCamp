Présentation générale

Nous avions à réaliser une plateforme de jeu en ligne constituée de différents éléments qui devaient pouvoir communiquer entre eux afin de respecter le diaggrame qui nous a été communiqué.

Dans un premier temps, nous avons établis un plan des différentes tâches à effectuer en fonction des missions, puis nous nous sommes répartis le travail en prenant en compte les compétences de chacun.

Nous avons choisi de diviser le projet en 6 groupes :

  • Conception des Manettes

  • Création des différents jeux de la plateforme

  • Configuration du Raspberry Pi pour héberger et faire communiquer les manettes avec les jeux

  • Création d’une API pour transmettre les données des jeux à la plateforme spectateurs, ainsi que toutes les informations liées à la partie sur l’application

  • Le mode spectateur pour permettre aux utilisateurs de suivre une partie en direct avec un affichage des scores

  • Création de deux applications; La première pour la gestion intégrale du projet et la deuxième pour pouvoir consulter l’historique des scores.

Tâches :

  • Revue de la liste des accessoires à incorporer pour la manette

  • Modélisation de la manette

  • Avoir une manette qui communique avec le jeu réseau

  • Incorporer les éléments dans la manette

Création des manettes de jeu

Le but de notre projet est de réaliser une manette connectée qui puisse communiquer avec les jeux.

Nous avons dans un premier temps réalisé la manette sur Arduino avec tous les composants qu'il nous fallait ( boutons, touches directionnelles, led).

Nous avons réalisé la création d’un plan 3D de la manette pour contenir les composants que l’on a imprimés en 3D.

Pour finir, nous nous sommes servis du protocole de messagerie MQTT pour pouvoir envoyer les informations de la manette.

Outils :

  • Carte arduino Wemos D1 R2

  • IDE arduino

  • Accessoire relatif à la carte (bouton poussoir, fils, LED)

  • Tinkercad pour les tests de branchements et la création du plan 3D

  • Imprimante 3D pour l’impression de la manette

Tâches :

  • Revue de la liste des accessoires à incorporer pour la manette

  • Modélisation de la manette

  • Avoir une manette qui communique avec le jeu réseau

  • Incorporer les éléments dans la manette

Plateforme pour les spectateurs

Notre objectif est de permettre la visualisation de la partie en cour, le nom du jeu, les scores, les pseudo des joueurs-joueuses et la durée de la partie en cours.

Pour cela nous avons créé un site hebergé sur Github, intégré une copie de chaque jeu non jouable depuis notre page, car le streaming est trop lourd pour le Raspberry Pi, et récupéré les données de la manette et de la plateforme de jeu afin de simuler la partie en cours.

Outils :

  • Maquettage de notre plateforme : Illustrator

  • Intégration de notre plateforme : HTML / SCSS (compilé en CSS) / JS

  • Récupération de données et mise à jour sur la page : JS (JQuery) / WebSocket

Tâches :

  • Maquettes de l’interface

  • Intégration de l’interface

  • Intégration du code source des jeux dans notre page web

  • Définition de quelles données on doit recevoir, sous quelle forme et quelle organisation.

  • Récupération des données de la partie (nom du jeu, pseudos, score et timer) et les afficher.

  • Récupération des données des manettes et simuler la partie en cours sur notre plateforme.

Gestion de projet et historique des parties jouées

Les objectifs de notre mission sont dans un premier temps la gestion du projet, ainsi que la création d’un historique de jeux.

Pour que chaque groupe puisse organiser et coordonner ses tâches, nous avons dans un premier temps créé une base de données sur Google Sheets, directement reliée à l’interface de Glide.

À partir de ça nous avons pu concevoir deux applications depuis une interface No code reliant directement les informations contenus dans les sheets sur GlideApps.

Les applications sont directements liées, grâce aux informations présentent sur la même google sheet.

Outils :
